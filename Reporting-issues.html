Reporting issues

After you confirmed that you have found a issue on a website is time to report!
Follow theses steps to do this:

  1. Click on the Report icon (or right click > Report site issue)
  2. The first question is: What seems to be the trouble?
    Select one of the options that describes the issue you found
  3. The next question is about details of the issue, they gave us a model that we can fill with information that will help finding the problem and getting to a solution.
  4. The webcompat add-on will fill the next questions with Site Url, Browser/version, Operating system and a screenchot of the website.

Reporting via Github Vs. Reporting Anonymously

When you answer all the questions related to the issue is time to submit the informations.
Now you have two options for do that: 1. Report via Github and 2. Report Anonimously.

Report via Github (recommended)
When you send a report via Github a issue will be created in the repo of webcompat and you can follow the discussion about the issue.
If they need more information they can talk with you and you can send complimentary information on the issue if you want to.
This will help either in the next step (site outreach).

Report Anonymously
You can also send the report anonymously, but in this case you will have to follow up the reported issue by yourself accessing the webcompat website (or the github issue they will create with your report).

After reporting, what else can I do?

After reporting a issue is time to wait, someone will check your report and people will try to reproduce your reported issue.
If they can reproduce the issue and identify that it's a real web compatibility issue they will start searching for solutions to the problem.

Sometimes the solution involves some action by the website developers, so they start a new phase: try to enter in contact with the website owners or the developers.
If you want, you can help in this too!
